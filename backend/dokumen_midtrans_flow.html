1. Ringkasan Eksekutif & Informasi Merchant

Dokumen ini disusun sebagai bagian dari pemenuhan syarat kelayakan pendaftaran dan verifikasi akun merchant Midtrans untuk aplikasi Creative Studio Kiosk Photobooth. Aplikasi ini merupakan sistem Kiosk Photobooth self-service berbasis Android dan Web Portal yang memungkinkan pengguna membeli sesi foto secara mandiri, mengendalikan kamera kiosk via smartphone mereka (remote controller), memotret secara otomatis, serta mengunduh hasil foto digital dan file timelapse (GIF) langsung ke ponsel mereka.

Untuk memfasilitasi transaksi pembayaran nontunai yang cepat, otomatis, dan aman di area kiosk, Creative Studio mengintegrasikan Midtrans Snap API dengan mode pembayaran utama berupa QRIS dan dompet digital ShopeePay. Proses pembukuan antrean dan sinkronisasi hardware kiosk dilakukan secara real-time berdasarkan notifikasi status sukses dari server Midtrans.

3. Penjelasan Langkah-Langkah Customer Journey (End-to-End)

Berikut adalah rincian detail operasional dari langkah-langkah yang dilalui oleh customer:

  1. Membuka Halaman Pemesanan (Pilih Paket): Customer membuka portal pemesanan Creative Studio (`order.php`) melalui peramban web ponsel atau layar tablet Kiosk. Mereka melihat katalog paket foto yang tersedia, antara lain: Cetak Receipt (Rp 15.000), Paket Cetak Warna (Rp 25.000), dan Paket ID Card (Rp 35.000). Setelah memilih salah satu paket, customer menekan tombol PESAN SEKARANG.
  2. Pembuatan Antrean di Database Lokal: Sistem backend memproses input formulir, menghasilkan pengenal sesi unik sepanjang 16 karakter acak (`session_id`), dan menetapkan nomor antrean berikutnya. Data transaksi dicatat ke file antrean server (`queue.json`) dengan status UNPAID. Selanjutnya, halaman dialihkan ke halaman gerbang pembayaran (`payment_gateway.php`).
  3. Pemuatan Halaman Checkout & Pemanggilan Snap API: Halaman `payment_gateway.php` memuat data transaksi. Jika metode pembayaran diset menggunakan Midtrans, server merchant secara otomatis melakukan panggilan HTTP POST (cURL) ke server Midtrans Snap API dengan payload berisi ID Transaksi (Order ID), nominal pembayaran, serta detail item paket. Panggilan ini terotentikasi secara aman menggunakan **Midtrans Server Key**.
  4. Interaksi Antarmuka Pembayaran Midtrans Snap (Pilihan ShopeePay): Sistem menerima respon berupa token transaksi dari Midtrans. Di sisi client, perpustakaan JavaScript Midtrans Snap (`snap.js`) mendeteksi token tersebut dan membuka modal popup secara otomatis. Customer dapat melihat berbagai saluran pembayaran yang aktif. Untuk melakukan pembayaran dengan ShopeePay, customer dapat menekan opsi QRIS (GoPay, ShopeePay, LinkAja) atau opsi langsung ShopeePay.
  5. Eksekusi Pembayaran ShopeePay:
    • Mobile Flow (Smartphone): Snap secara otomatis membuka aplikasi Shopee di ponsel customer melalui tautan dalam (deeplink). Aplikasi Shopee akan menampilkan halaman konfirmasi pembayaran Creative Studio dengan nominal yang sesuai. Customer menekan tombol bayar dan memasukkan PIN ShopeePay mereka.
    • Desktop/Tablet Flow (Layar Kiosk): Layar menampilkan kode QR dinamis (QRIS/ShopeePay). Customer membuka aplikasi Shopee di ponsel mereka, menekan tombol Scan, memindai kode QR di layar, mengonfirmasi nominal, dan memasukkan PIN ShopeePay.
  6. Penerimaan Webhook Notifikasi & Validasi Signature Key: Seketika setelah PIN ShopeePay berhasil diverifikasi di server Shopee/Midtrans, Midtrans mengirimkan notifikasi callback HTTP POST secara asynchronous ke server merchant. Backend PHP merchant menerima payload notifikasi, menghitung hash SHA-512 (Signature Key) lokal menggunakan Server Key untuk menjamin keaslian data, dan memverifikasi status transaksi. Jika status transaksi bernilai settlement atau capture, status pesanan di database lokal diubah menjadi WAITING (siap antre) atau ACTIVE (mulai sesi foto).
  7. Polling Status & Pengalihan Halaman (Redirection): Secara paralel, halaman checkout customer melakukan polling AJAX ke server merchant setiap 4 detik untuk memeriksa pembaruan antrean. Begitu notifikasi sukses terverifikasi, halaman checkout customer akan menampilkan konfirmasi sukses dan mengalihkan halaman secara otomatis ke Halaman Remote Controller Kiosk (`order.php?session_id=`).
  8. Penggunaan Kiosk Kios Foto (Mulai Sesi): Di halaman remote controller, customer kini dapat memilih layout foto (Strip/Grid/Postcard) dan memilih tema bingkai (frame). Ketika customer menekan tombol merah besar MULAI FOTO, server mengirim perintah tangkap (`START_CAPTURE`) ke kamera kiosk. Kamera kiosk mengambil pose, menggabungkan foto dengan bingkai pilihan, dan mengunggah hasilnya. Customer dialihkan ke portal unduhan (`index.php?id=`) untuk menyimpan berkas foto strip berkualitas tinggi ke perangkat mereka.

5. Mekanisme Verifikasi Keamanan Notifikasi (Signature Key)

Untuk menjamin keamanan pertukaran data transaksi dan mencegah penipuan manipulasi status pembayaran (seperti parameter status pembayaran palsu), server Creative Studio menerapkan validasi ketat terhadap setiap callback/notifikasi webhook yang dikirim oleh Midtrans menggunakan algoritma hashing SHA-512.

Setiap kali status pembayaran diperbarui, server Midtrans akan mengirimkan payload JSON ke url callback merchant. Server PHP merchant akan menangkap parameter order_id, status_code, gross_amount, dan signature_key. Server kemudian melakukan kalkulasi mandiri menggunakan kunci rahasia (Server Key) yang hanya diketahui oleh merchant dan Midtrans:

$localSignature = hash("sha512", $orderId . $statusCode . $grossAmount . $serverKey);

Jika nilai hash $localSignature yang dihitung di sisi merchant cocok secara identik dengan signature_key yang dikirim oleh Midtrans di dalam header/body callback, maka transaksi dianggap valid dan aman. Sistem akan mengubah status pembayaran menjadi lunas. Sebaliknya, apabila tanda tangan digital tidak cocok, server merchant akan menolak callback dengan kode HTTP 403 Forbidden dan membatalkan aktivasi antrean sesi foto.

💡 Catatan Keandalan Sistem:
Selain callback asinkron dari Midtrans, checkout page Creative Studio juga dilengkapi sistem pemantau berkala (client polling) yang memanggil status transaksi langsung ke API Midtrans Status endpoint: https://api.sandbox.midtrans.com/v2/{order_id}/status. Metode ganda ini menjamin pengguna mendapatkan respon instan (real-time redirect) begitu mereka menyelesaikan otorisasi ShopeePay.

6. Pernyataan Kepatuhan Merchant

Creative Studio menjamin bahwa seluruh data pelanggan dan proses otorisasi dompet digital ShopeePay dilakukan secara terenkripsi menggunakan protokol HTTPS aman. Sistem merchant tidak menyimpan data kredensial pembayaran sensitif seperti PIN e-wallet, nomor kartu kredit, atau data autentikasi langsung pengguna. Seluruh proses checkout didelegasikan secara penuh ke infrastruktur pembayaran Midtrans yang telah tersertifikasi PCI-DSS Level 1.
